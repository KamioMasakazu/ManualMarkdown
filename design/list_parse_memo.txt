markdownのリスト行を取り出してパースする関数。

インターフェースは次の通り。
```
def is_list_lines(self, lines: deque[str]) -> ListData | None:
```

仕様は次の通り。
linesはファイルのデータが1行ずつ入っている。
先頭行がmarkdownのリスト行でなければNoneを返す。
先頭行がmarkdownのリスト行ならそのリストの終わりまでをlinesから取り出してローカル変数list_lines（deque型）に移す。（lines.popleft()で破壊的に操作する）。
リストの終わりには空行がある。この空行はpopleft()しない。
list_linesの各行のインデントがタブならスペースに変換する。

取り出した行をリストの入れ子構造を考慮してLineData型の値にする。これはヘルパ関数を作って行う。
ヘルパ関数のインターフェースは次の通り。
```
def _parse_lines(self, list_lines: deque[str], current_indent: int = 0) -> ListData:
```
is_list_lines()が少なくとも1行はリストにできる行を渡すのでListDataを返せないことはない。
current_indentは行頭のインデントが変化したときの行が-や1.で始まる（リスト要素の始まりである）ときに、子のリストにするか（インデントが深くなった時）、戻るかの判断に使用する。


LineDataは次の通りである。
```
@dataclass(frozen=True)
class ListData:
	type: str	# ulかolか
	lines: list[list[str] | ListData]	# <li>から</li>の内容 
```

ListData::typeはリスト形式に応じて"ul"か"ol"である。文字列で良い。
ListData::linesはHTMLにしたとき<li>〜</li>の内容がlistの1要素になる。
<li>〜</li>が複数行になる時、つまりmarkdownが次の様な時はlist[str]が要素になる。
```
- 何か文章
  続きの行
  ...
- 2つ目のリスト要素
```
これは
```
ListData(
	type = "ul",
	lines = [
		["何か文章", "続きの行", "..."],
		["2つ目のリスト要素"]
	]
)
```
になる。
リストが入れ子になっているときは子孫要素のListDataを要素に持つ。
```
- 1行目の要素
  何か文章
  1. 入れ子の1つ目の要素
    入れ子の1つ目の要素の続きの文章
	...
  2. 入れ子の2つ目の要素
    入れ子の1つ目の要素の続きの文章
	...
- 2行目の要素
```
これは次の様になる。
```
ListData (
	type = "ul",
	lines = [
		[
			"1行目の要素",
			"何か文章",
			ListData(
				type = "ol",
				lines = [
					[
						"入れ子の1つ目の要素",
						"入れ子の1つ目の要素の続きの文章",
						"..."
					],
					[
						"入れ子の2つ目の要素",
						"入れ子の2つ目の要素の続きの文章",
						"..."
					]
				]
			)
		],
		["2行目の要素"]
	]
)
```
