# ManualMarkdown仕様書
標準的なMarkdwon -> HTML変換に独自書式を追加する。
Pythonで実装する。
利用可能なライブラリがあるなら使用してよい。

## コマンドライン
ManualMarkdown.py <input.md> [-o path/to/input.html] [--css path] [--js path] [--embed-image]

input.md
必須
入力となるマークダウンファイル。
文字コードはUTF-8である。

-o
HTML出力先ファイル名。
ディレクトリがなければ作る。
未指定時は標準出力に出力する。

--css
デフォルト以外のcssを使用する場合に指定する。
ファイルもしくはディレクトリの指定。
ディレクトリを指定する場合はそのディレクトリにあるcssファイルすべてが対象になる。
未指定の場合はManualMarkdown.pyと同じディレクトリにあるdefault.cssを使う。
cssの内容はHTMLのstyleタグに展開する。linkはしない。
styleタグはscriptタグよりも前に配置する。

--js
javascriptを利用する場合に指定する。未指定なら無視する。
ファイルもしくはディレクトリの指定。
ディレクトリを指定する場合はそのディレクトリにあるjsファイルすべてが対象になる。
javascriptのコードはscriptタグに展開する。hrefで参照しない。

--embed-image
画像ファイルをimgタグのsrc属性に"data:..."で埋め込む。
MIMEタイプは拡張子から推定する。
未指定時はファイルをコピーする。詳細は「## その他 - ### 画像ファイル」を参照。


## 拡張書式
### 種別指定ブロック
次のmarkdown書式を

````
```@box:種別指定[タイトル]
本文・・・
本文・・・
```
````

次のHTMLにする。
```
<div class="種別指定">
  <div>タイトル</div>
  <div>
    <p>本文・・・</p>
    <p>本文・・・</p>
  </div>
</div>
```
「```@box:種別指定[タイトル]」のブロック内はMarkdownとして解釈する。
種別指定はinfo、notice、...と任意に設定でき、divタグのクラスになる。使用可能な文字は半角英数字、アンダースコア、マイナス記号である。
ネスとする場合は````で囲むこと。

### CSVテーブル
CSVの内容をテーブルにする。
CSVの解釈はPython標準csv（excel方言相当）に従う。

#### 外部ファイル
```
@csv[:row_th | :col_th](path/to/file)
```

path/to/fileで指定されたcsvファイルの内容をテーブルにする。パスは入力したMarkdownのあるディレクトリを起点とする。

注意: 外部ファイル指定は「コードブロック内の1行」に書く。
（コードブロック外に書いた場合は通常テキストとして扱う）

[カッコ]は省略可能の意味である。したがって指定は@csv、@csv:row_th、@csv:col_thのいずれか。
@ではなく@csvが指定子である。
@csv:row_thは先頭行がthであることを意味する。
@csv:col_thは先頭列がthであることを意味する。
@csvならすべてtdである。

#### インライン定義
````
```@csv[:row_th | :col_th]
a,b,c,...
d,e,f,...
...
```
````

「```@csv[:row_th | :col_th]」のブロック内はCSVデータである。
コードブロック内をテーブルにする。
その他は「外部ファイル」指定の場合と同じ。

## HTMLの構成
出力されるHTMLは適切にインデントする。
全体をarticleとし、見出しごとにsectionにする。
sectionは入れ子になる。つまり、```<h1>```を含むセクションは```<h2>```を含むセクションを子要素とし、```<h2>```を含むセクションは```<h3>```を含むセクションを子要素とし、・・・となる。

```
# 見出し1
本文・・・

## 見出し2
本文2・・・
```

これは次のようなHTMLになる。

```
<article>
  <section>
    <h1>見出し1</h1>
    <p>本文・・・</h1>
    <section>
      <h2>見出し2</h2>
      <p>本文2・・・</p>
    </section>
  </section>
</article>
```

## その他
### デフォルトで改行までが段落。
通常のテキスト行は1行ごとに段落（```<p>```）とする。ただしリスト、引用、コードブロック等のブロック要素内ではMarkdown標準の解釈を優先する。

### 画像ファイル
Markdownの画像ファイルのパスは入力ファイルのあるディレクトリを基準として解決する。
出力先が入力mdと同一ディレクトリの場合：画像はコピーしない（既存参照をそのまま使う）
出力先が別ディレクトリの場合：assetsへコピーする
assetsへコピーする際、同名ファイルが既に存在する場合はコピーしない（上書きしない）
その結果、参照先ファイルが存在しない場合は警告を標準エラーに出力する。
